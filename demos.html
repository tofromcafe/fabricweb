Interactive Demos: Try Fabric.js Yourself!

Hands-on examples showing how to use Fabric.js for common canvas operations. Perfect for beginners to understand how the code translates into visual results.

üéì New to Programming?

Don't worry! These demos show you exactly how a few lines of JavaScript code can create interactive graphics. Each demo includes:

  • Live Canvas - See the results immediately
  • Code Examples - The exact JavaScript that makes it work
  • Step-by-Step - What each line of code does
  • Try It - Buttons to experiment with different options

Demo 1: Adding Text to Canvas

How It Works

// 1. Create a canvas (like a digital drawing board) const canvas = new fabric.Canvas('textCanvas'); // 2. Create a text object (like typing on the canvas) const text = new fabric.Text('Hello Fabric.js!', { left: 50, // Position from left edge top: 100, // Position from top edge fontSize: 20, // Size of the text fill: 'blue' // Color of the text }); // 3. Add the text to the canvas (make it visible) canvas.add(text);

üéØ What's Happening?

Think of this like placing a sticker on a bulletin board. The canvas is your board, and the text is your sticker that you can move around, resize, or edit.

üìê Coordinates

left: 50, top: 100 means "place this 50 pixels from the left edge and 100 pixels from the top." It's like giving someone directions!

Demo 2: Adding a Yellow Rectangle

How It Works

// 1. Create a rectangle object (like drawing a box) const rect = new fabric.Rect({ left: 100, // X position (horizontal) top: 50, // Y position (vertical) width: 120, // How wide the box is height: 80, // How tall the box is fill: 'yellow', // Fill color (inside the box) stroke: 'orange', // Border color strokeWidth: 3 // How thick the border is }); // 2. Add the rectangle to the canvas canvas.add(rect);

üé® Fill vs Stroke

Fill is like paint bucket - fills the inside. Stroke is like a pen - draws the outline. You can have one, both, or neither!

üìè Size & Position

Width and height are in pixels (tiny dots on your screen). Position (left, top) is measured from the top-left corner of the canvas.

Demo 3: Changing Canvas Background

How It Works

// 1. Simple solid color background canvas.setBackgroundColor('yellow', canvas.renderAll.bind(canvas)); // 2. Or create a gradient background (fancy!) const gradient = new fabric.Gradient({ type: 'linear', coords: { x1: 0, y1: 0, x2: 400, y2: 300 }, colorStops: [ { offset: 0, color: '#ffeb3b' }, // Start: bright yellow { offset: 1, color: '#ff9800' } // End: orange ] }); canvas.setBackgroundColor(gradient, canvas.renderAll.bind(canvas));

üé® renderAll() Function

Think of this like hitting "refresh" on your browser. It tells the canvas to redraw everything so you can see the changes you made.

üåà Gradients Explained

A gradient smoothly blends from one color to another. coords tells it where to start and end the blend. colorStops are the colors to blend between.

Understanding the Programming Concepts

üèóÔ∏è Objects & Classes

Think of it like cookie cutters:

  • Class = The cookie cutter shape (like fabric.Rect)
  • Object = The actual cookie you make (like new fabric.Rect())
  • You can make many cookies from one cutter, each with different decorations (properties)

üéõÔ∏è Properties & Methods

Like features on a TV remote:

  • Properties = Settings you can change (volume level, channel)
  • Methods = Buttons you can press (turn on, change channel)
  • rect.fill = 'yellow' changes a setting
  • canvas.add(rect) presses a button

üìö APIs (Application Programming Interfaces)

Like a restaurant menu:

  • The menu shows what you can order (available functions)
  • You don't need to know how to cook it (internal implementation)
  • You just say "I want a burger" (call the function)
  • Fabric.js API lets you say "I want a rectangle" without knowing how pixels work

üé™ Event-Driven Programming

Like a doorbell system:

  • You set up listeners (install the doorbell)
  • Events happen (someone presses the button)
  • Your code responds (you answer the door)
  • Mouse clicks, key presses, object movements are all "doorbell rings"
