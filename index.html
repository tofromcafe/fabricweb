What is Fabric.js?

Fabric.js is an object model for HTML5 canvas that enables interactive object manipulation on the canvas. It provides a comprehensive suite of tools for:

Core Architecture Overview

graph TD A[Static Canvas] --> B[Selectable Canvas] B --> C[Canvas] D[FabricObject] --> E[Interactive Object] E --> F[Shape Objects] F --> G[Rect] F --> H[Circle] F --> I[Text] F --> J[Image] C --> K[Event System] C --> L[Rendering Engine] M[Layout Manager] --> N[Active Selection] M --> O[Group Layout] P[Filters] --> Q[Image Filters] P --> R[Color Filters]

Fabric.js follows a hierarchical object model where everything inherits from base classes, providing consistent behavior across all canvas objects.

Key Concepts for Beginners

üéì New to Programming?

Don't worry! Here are some key terms you'll see throughout this documentation:

API: Application Programming Interface - like a menu at a restaurant that shows what you can order (what functions you can use)
Object: A bundle of data and functions that work together - like a digital "thing" with properties and abilities
Method: A function that belongs to an object - something the object can "do" (like a dog can bark)
Property: A piece of information about an object - something the object "has" (like a dog has a color)
Event: Something that happens that your code can respond to (like a mouse click or key press)
Canvas: A digital drawing surface where you can place and manipulate visual objects

Canvas Hierarchy

Everything starts with a Canvas instance that manages objects and handles rendering. Objects are organized in layers and can be grouped for collective manipulation.

Object Model

All drawable items inherit from FabricObject, providing common properties like position, size, rotation, and styling options.

Event-Driven

User interactions trigger events that can be captured and handled, enabling rich interactive experiences.

Transformation Matrix

Mathematical transformations handle object positioning, scaling, rotation, and skewing in an efficient way.
